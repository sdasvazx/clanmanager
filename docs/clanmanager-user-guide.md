# 클랜 매니저 사용설명서

이 문서는 `clanmanager` 앱에서 **출석 기록 → 참여율/참여점수 계산 → 분배금 계산 → 금고 지급**까지 이어지는 전체 흐름을 공유하기 쉽게 정리한 문서입니다.

---

## 1. 전체 흐름

```text
출석체크 / 보스 참여 등록
        ↓
출석보스 설정 기준으로 활동별 참여점수·패널티 계산
        ↓
참여율 조회 / 내 정보 확인 / 로비 순위 반영
        ↓
분배금 조회에서 회차·컷·분배 다이아 입력
        ↓
참여분배 + 투력분배 계산
        ↓
분배금 적립
        ↓
금고 잔액 차감 + 회원별 미수령 분배금 생성
        ↓
마이페이지에서 본인이 수령 확인
```

---

## 2. 출석·참여율 시스템

### 2-1. 출석보스 설정

관리자 설정의 **출석보스 설정**에서 참여율과 참여점수 계산에 들어갈 활동을 관리합니다.

| 항목 | 설명 |
|---|---|
| 활동명 | 예: `13시 (2성)`, `17시 (1성)`, `21시 (2성)`, `소수쟁` |
| 참여점수 | 해당 활동에 참석했을 때 받는 기본 점수 |
| 패널티 적용 여부 | 미참여 시 감점할지 여부 |
| 미참여 패널티 점수 | 패널티 ON일 때 빠진 경우 차감되는 점수 |
| 표시 순서 | 참여율 조회 표에 표시되는 순서 |
| 활성 여부 | 비활성/삭제된 활동은 계산과 표에서 제외 |

기본 활동은 아래 10개입니다.

1. 소수쟁
2. 13시 (2성)
3. 17시 (1성)
4. 21시 (2성)
5. 결승전
6. 마슈미드
7. 에노크
8. 전초전
9. 클랜임무
10. 클랜수호

### 2-2. 참여율 계산

참여율은 **점수 기준이 아니라 참석 횟수 기준**으로 계산합니다.

```text
참여율 = 본인이 참석한 계산 대상 활동 수 / 전체 계산 대상 활동 수 × 100
```

비활성화된 활동은 전체 횟수와 참석 횟수에서 모두 제외됩니다.

### 2-3. 참여점수 계산

```text
기본 참여점수 = 참석한 활동의 참여점수 합계
미참여 패널티 = 패널티 ON인 활동에 빠졌을 때 차감되는 점수 합계
소수쟁 가산점 = 소수쟁 화면에서 별도로 지급한 가산점

최종 참여점수 = max(0, 기본 참여점수 - 미참여 패널티 + 소수쟁 가산점)
```

최종 참여점수는 **0점 밑으로 내려가지 않습니다.**

### 2-4. 회차 시스템

참여율과 참여점수는 회차 기준으로 볼 수 있습니다. 보통 2주 단위로 운영합니다.

예시:

- 2026-07-08 ~ 2026-07-22
- 2026-07-22 ~ 2026-08-05

분배금 조회에서는 **분배 회차**를 선택해야 해당 회차 기준 참여점수가 제대로 계산됩니다.

## 3. 보스 참여내역 조회 / 출석체크

### 3-1. 보스 참여내역 조회

이미 등록된 보스 참여 기록을 확인하는 화면입니다.

확인 가능한 정보:

- 날짜
- 컷시간
- 출석 입력 시간
- 보스명
- 참석 인원
- 클랜별 인원
- 참여 명단

### 3-2. 출석체크

관리자가 실제 보스 스크린샷을 올려 출석 인원을 등록하는 화면입니다.

주요 기능:

- 사진 업로드
- OCR 이름 인식
- 클랜별 자동 분류
- 잘못 인식된 이름 수정
- 누락 인원 수동 추가
- 오인식 후보 확인
- 참여 명단 저장

저장 시 활동명이 출석보스 설정의 활동과 연결되면 `ActivityAttendance` 출석 기록이 생성되고, 참여율과 참여점수에 반영됩니다.

## 4. 투력점수 시스템

### 4-1. 투력점수 공식

```text
투력점수 = 성장점수 + 현재투력점수
```

현재투력은 관리자가 회원별로 입력합니다.

### 4-2. 구간별 점수

| 투력 구간 | 만당 점수 |
|---|---:|
| 0만 ~ 80만 | 1점 |
| 80만 ~ 90만 | 1점 |
| 90만 ~ 95만 | 2점 |
| 95만 ~ 100만 | 3점 |
| 100만 ~ 105만 | 5점 |
| 105만 ~ 110만 | 7점 |
| 110만 ~ 115만 | 9점 |
| 115만 이상 | 12점 |

예시: 현재투력 92만

- 0~80만: 80점
- 80~90만: 10점
- 90~92만: 2만 × 2점 = 4점
- 현재투력점수 = 94점

### 4-3. 성장점수

이전투력에서 현재투력까지 증가한 만큼을 구간별로 계산합니다.

예시: 85만 → 92만

- 85만 → 90만: 5만 × 1점 = 5점
- 90만 → 92만: 2만 × 2점 = 4점
- 성장점수 = 9점

최종 투력점수:

```text
성장점수 9점 + 현재투력점수 94점 = 103점
```

## 5. 분배금 조회 시스템

### 5-1. 분배 모드

| 모드 | 설명 |
|---|---|
| 클랜별 분배 | 귀신 / 운좋은 / 귀신Z / 로망 클랜별로 다이아를 따로 입력 |
| 전체 분배 | 클랜 구분 없이 전체 인원 대상으로 계산 |

### 5-2. 참여분배와 투력분배

분배금은 **참여분배 다이아**와 **투력분배 다이아**를 따로 입력합니다.

각각 독립적으로 계산됩니다.

```text
참여분배 = 참여율 컷을 넘은 사람끼리 최종 참여점수 비율로 배분
투력분배 = 투력점수 컷을 넘은 사람끼리 투력점수 비율로 배분

최종 지급액 = 참여분배 + 투력분배
```

참여율 컷을 못 넘어도 투력점수 컷을 넘으면 투력분배는 받을 수 있습니다. 반대로 투력점수 컷을 못 넘어도 참여율 컷을 넘으면 참여분배는 받을 수 있습니다.

### 5-3. 분배금 계산 시 회차 선택

분배금 계산 시 **분배 회차**를 선택합니다.

`전체 기간`은 테스트용에 가깝고, 실제 운영에서는 특정 회차를 선택하는 것을 권장합니다.

### 5-4. 소수점 처리

개인별 지급액은 소수점이 생기면 버림 처리합니다.

```text
개인지급액 = floor(점수 × 1점당 다이아)
```

그래서 소수점 버림으로 남는 다이아가 생길 수 있습니다.

| 표시명 | 의미 |
|---|---|
| 실제 지급 합계 | 회원들에게 실제 배정된 총 다이아 |
| 미배분 | 소수점 버림 등으로 아직 배정되지 않고 남은 다이아 |

### 5-5. 히스토리 저장

히스토리 저장 시 아래 내용이 함께 저장됩니다.

- 분배 모드
- 선택 회차
- 참여율 컷
- 투력점수 컷
- 클랜별 참여분배 다이아
- 클랜별 투력분배 다이아
- 회원별 지급액
- 저장 시각

## 6. 금고 시스템

### 6-1. 분배금 적립

분배금 적립 버튼을 누르면 계산된 회원별 지급액이 기존 금고 시스템으로 넘어갑니다.

동작 순서:

1. 회원별 지급액 계산
2. 금고 잔액 확인
3. 금고 잔액이 부족하면 오류 표시
4. 충분하면 금고 잔액 즉시 차감
5. 회원별 미수령 분배금 생성

금고 잔액은 **분배금 적립 시점에 줄어듭니다.**

### 6-2. 회원 수령 확인

회원은 마이페이지에서 본인의 미수령 분배금을 확인하고 수령 처리할 수 있습니다.

이 기능은 실제 자동 송금이 아니라 운영 기록용입니다.

### 6-3. 동시성 보호

금고에는 동시 저장 충돌을 막기 위한 낙관적 락이 적용되어 있습니다.

기존 금고 데이터의 `version` 값이 비어 있어도 오류가 나지 않도록 기본형 `long`을 사용합니다.

## 9. 최근 해결된 주요 이슈

| 이슈 | 처리 내용 |
|---|---|
| 출석보스 설정 저장 실패 | CORS에 `PUT` 허용 추가 |
| 참여점수 음수 발생 | 최종 참여점수를 `0점` 미만으로 내려가지 않게 처리 |
| 분배금 계산이 전체 기간 기준으로 섞임 | 분배 회차 선택 기능 추가 |
| 분배금 반올림으로 과지급 가능 | 반올림 대신 버림 처리 |
| 금고 적립 시 `version NULL` 오류 | `ClanVault.version`을 기본형 `long`으로 변경 |
| 참여율/분배 계산 풀스캔 우려 | `ActivityAttendance(attendance_date, status)` 인덱스 추가 |
| 탈퇴/비활성 회원 로그인 가능 | 로그인 시 활성 회원만 허용 |
| 기타설정 클래스 선택 문제 | 클랜/클래스 선택 저장 로직 정리 |

## 10. 운영 시 주의사항

### 10-1. 분배 전 확인

분배금 적립 전에는 아래 항목을 확인하는 것이 좋습니다.

- 분배 회차가 올바른지
- 참여율 컷이 맞는지
- 투력점수 컷이 맞는지
- 클랜별 참여분배 다이아가 맞는지
- 클랜별 투력분배 다이아가 맞는지
- 금고 잔액이 충분한지
- 회원별 지급액이 이상하지 않은지

### 10-2. 출석 등록 시 확인

OCR 인식은 편의 기능이므로 저장 전 확인이 필요합니다.

- 누락 인원이 없는지
- 잘못 인식된 닉네임이 없는지
- 기준 클랜이 맞는지
- 보스명/활동명이 출석보스 설정과 연결되는지
- 날짜와 시간이 맞는지

## 11. 용어 정리

| 용어 | 의미 |
|---|---|
| 참여율 | 참석 횟수 / 전체 계산 대상 활동 수 |
| 기본 참여점수 | 참석한 활동의 점수 합계 |
| 미참여 패널티 | 패널티 ON 활동에 빠졌을 때 차감되는 점수 |
| 소수쟁 가산점 | 소수쟁 화면에서 별도로 지급한 점수 |
| 최종 참여점수 | 기본 참여점수 - 패널티 + 소수쟁 가산점 |
| 기여율 | 최종 참여점수 기준 기여 비율 |
| 투력점수 | 성장점수 + 현재투력점수 |
| 참여분배 | 참여율 컷 통과자끼리 참여점수 비율로 나누는 다이아 |
| 투력분배 | 투력점수 컷 통과자끼리 투력점수 비율로 나누는 다이아 |
| 실제 지급 합계 | 실제 회원들에게 배정된 총 다이아 |
| 미배분 | 소수점 버림 등으로 남은 다이아 |
| 미수령 분배금 | 적립은 되었지만 회원이 아직 수령 확인하지 않은 금액 |

## 12. 한 줄 요약

클랜 매니저는 **출석을 정확히 기록하고, 회차별 참여율과 참여점수를 계산한 뒤, 참여분배와 투력분배를 분리해서 공정하게 다이아를 분배하고 금고에 기록하는 관리 시스템**입니다.
